Documentație Tehnică: Aplicație Web pentru Managementul Matricilor, Grafurilor, Șirurilor și Tablourilor

1. Introducere

Proiectul este o aplicație web dezvoltată în PHP care oferă utilizatorilor posibilitatea de a genera, salva, încărca și șterge structuri de date precum: matrici, grafuri, șiruri de caractere și tablouri (arrays). Interfața web este modernă, cu accent pe nuanțe de verde, iar aplicația include autentificare, roluri și un panou de administrare.

2. Arhitectură Generală

  • Frontend: HTML + CSS (fișiere dedicate pentru fiecare modul, stil unificat verde)
  • Backend: PHP procedural (fără framework), cu module distincte pentru fiecare funcționalitate
  • Persistență: MySQL (bază de date cu două tabele: users și generated_data)
  • Autentificare: JWT transmis în headerul Authorization fără utilizarea cookie-urilor

3. Funcționalități

3.1 Autentificare și înregistrare

  • Utilizatorii se pot înregistra cu username, email și parolă
  • Parola este criptată cu password_hash()
  • Autentificarea generează un JWT
  • Tokenul JWT este transmis ulterior în headerul HTTP Authorization: Bearer <token>

3.2 Module funcționale

  • Matrici: generare aleatorie, salvare, încărcare, ștergere
  • Grafuri: definire prin vector de tați, muchii sau matrice de adiacență + SVG
  • Șiruri: generare, salvare, ștergere, afișare în cod HTML
  • Tablouri (arrays): funcționalități similare cu matricile

4. Panou de Administrare

Administratorii pot:

  • Vizualiza toți utilizatorii
  • Schimba roluri (user/admin)
  • Reseta parole
  • Accesa datele salvate de utilizatori

5. Interfață

  • Design modern și coerent cu temă verde (vezi style.css, admin.css, etc.)
  • Fiecare modul are o interfață distinctă și intuitivă
  • Interacțiuni asincrone minime, în general paginile sunt server-rendered

6. Baza de date

  • users: id, username, email, parola, rol
  • generated_data: id, user_id, titlu, data_type, data_json, created_at

Tipurile acceptate: matrix, string, graph, array

7. Securitate

  • Parolele sunt criptate
  • Accesul la date este restricționat pe baza rolului
  • Utilizatorii nu pot accesa datele altora
  • Toate interogările folosesc PDO + prepared statements

8. Arhitectura Sistemului - Model C4

Nivelul 1 - Context

Utilizatorii (anonimi sau autentificați) interacționează cu aplicația PHP. Backend-ul interacționează cu o bază de date MySQL.

@startuml C4_Context
!include https://raw.githubusercontent.com/plantuml-stdlib/C4-PlantUML/master/C4_Context.puml

Person(user, "Utilizator", "Folosește aplicația pentru a genera și gestiona structuri de date")
System(webapp, "Aplicația Web PHP", "Aplicație pentru matrici, grafuri, șiruri și tablouri")
System_Ext(mysql, "MySQL", "Stochează utilizatori și date")

Rel(user, webapp, "Interacționează prin interfața web")
Rel(webapp, mysql, "Citește/Scrie date")
@enduml

Nivelul 2 - Containere

@startuml C4_Containers
!include https://raw.githubusercontent.com/plantuml-stdlib/C4-PlantUML/master/C4_Container.puml

Person(user, "Utilizator")

System_Boundary(c1, "Aplicația Web PHP") {
  Container(frontend, "Frontend", "HTML/CSS/JS", "Interfață utilizator")
  Container(backend, "Backend", "PHP", "Logica aplicației și API-uri")
  ContainerDb(db, "MySQL", "Bază de date", "Persistă utilizatori și date generate")
}

Rel(user, frontend, "Utilizează prin browser")
Rel(frontend, backend, "Trimite cereri HTTP")
Rel(backend, db, "Interacționează prin PDO")
@enduml

Nivelul 3 - Componente

@startuml C4_Components
!include https://raw.githubusercontent.com/plantuml-stdlib/C4-PlantUML/master/C4_Component.puml

Container(backend, "Backend PHP", "PHP", "Conține logica aplicației și API-urile")

Component(auth, "Autentificare", "login_api.php, register_api.php", "Înregistrare și autentificare cu JWT")
Component(ui, "Interfață Module", "graphs.php, arrays.php, strings.php, matrices.php", "Pagini pentru manipulare structuri")
Component(storage, "Manipulare date", "save_*.php, load_*.php, delete_*.php", "Salvare și încărcare date în MySQL")
Component(admin, "Administrare utilizatori", "admin_users.php, change_role.php, reset_password.php", "Gestionare utilizatori")

Rel(auth, storage, "Validează utilizatorul pentru acces la date")
Rel(ui, storage, "Trimite cereri de salvare/încărcare")
Rel(admin, storage, "Accesează și modifică date administrative")
Rel(admin, auth, "Verifică și modifică conturi")
@enduml

9. Concluzii

Aplicația oferă o platformă educațională utilă pentru a înțelege și manipula structuri de date în mod vizual și persistent. Este ușor extensibilă și oferă o separare clară între componentele aplicației.
